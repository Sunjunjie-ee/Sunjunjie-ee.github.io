第141题考察点是判断环形链表，第142道题是判断环形的入口点。
这三道题都涉及到了环形链表知识点，针对这类问题只需要熟记两次双指针即可，第一次为快慢指针，快指针的步长是慢指针的二倍，当快慢指针第一次相遇时停止。第二次的双指针步数相同。下方列出判断环形的入口的核心代码：

ListNode fast = head, slow = head;
//第一次快慢指针
while(true){
    if(fast == null || slow == null) return null;
    fast = fast.next.next;
    slow = slow.next;
    if(fast == slow) break;
}
fast = head;
//第二次普通双指针
while(slow != fast){
    slow = slow.next;
    fast = fast.next;
}
return fast;

剑指offer52寻找两个链表的第一个公共节点问题可以转换成环问题的变种，取双指针分别在两个链表上遍历，当一个链表遍历完成后前往下一个链表，如果两个链表有公共节点的话，就一定会有相等的时候，代码如下：

public ListNode getIntersectionNode(ListNode headA, ListNode headB){
  ListNode nodeA = headA, ListNode nodeB = headB;
  while(nodeA != nodeB){
    //如果nodeA不为空就继续前往下一个节点，如果为空就前往B链表的头部节点
    nodeA = nodeA != null ? nodeA.next : nodeB;
    nodeB = nodeB != null ? nodeB.next : nodeA;
  }
  return nodeA
}

注意：并不用担心没有公共节点的时候会进入死循环的问题，因为从某种意义上来说，null也是公共节点。当nodeA和nodeB同时为null时循环就会结束了并返回null值。

2.1.2 非递归遍历

本处开始讨论树的遍历非递归实现。

1). 以前序遍历为例，先以根左右的顺序访问根节点，对于任意节点node，先访问其本身，之后判断左子树是否为空，如果不为空则重复上述步骤直到为空为止。当左子树为空时则开始访问右子树，由于访问左子节点之后需要反过来访问其右子节点所以此处借用栈的支持。

//前序遍历的非递归实现
void preOrderStack(TreeNode root){
    LinkedList<TreeNode> stack = new LinkedList<>();
    TreeNode curr = root;
    while(curr != null || !stack.isEmpty()){
        if(curr != null){
            System.out.print(curr.val);
            stack.push(curr);
            curr = curr.left;
        }else{
            TreeNode pre = stack.pop();
            curr = pre.right;
        }
    }
}

2). 层次遍历，层次遍历不同于上述三种遍历方式，层次遍历以每一层的节点为单位进行遍历。层序遍历的代 码较为简单，利用队列的FIFO特性，只需要一个队列即可，现在队列中加入根节点。之后对于任意一个节点 而言，出队列时如果其左子节点于右子节点有不为空的就将其放入队列。

//二叉树的层次遍历
public void levelRecur(TreeNode root){
  if(root == null) return null;
  LinkedList<TreeNode> queue = new LinkedList<>();
  //先将根节点加入队列
  queue.offer(root);
  while(!queue.isEmpty()){
    //将当前队列头部节点取出
    TreeNode node = queue.poll();
    System.out.print(node.val);
    //如果左子节点不为空就将左子节点放入到队列中
    if(node.left != null) queue.offer(node.left);
    //如果右子节点不为空就将右子节点放入到队列中
    if(node.right != null) queue.offer(node.right);
  }
}

剑指offer07

剑指offer07题明面上考察的是二叉树的重建，实质上其实还是树的深度遍历问题。只要有中序遍历以及前序遍历，后序遍历中的一个，就可以实现二叉树的重建。需要注意的是中序遍历是必须的，我们需要从中序遍历中得知根节点左右子树的位置，再从前序遍历或后序遍历中获取根节点的位置。本题给出的是前序遍历与中序遍历，示例代码如下：

HashMap<Integer, Integer> map = new HashMap<>();
int[] preOrder;
public TreeNode buildTree(int[] preorder, int[] inorder){
  this.preorder = preorder;
  //将中序遍历的元素放入哈希表中
  for(int i = 0; i < inorder.length; i++){
    map.put(inorder[i], i);
  }
  //三个参数分别为根节点索引（根据前序获得），左子树起始位置与右子树结束位置（根据中序遍历获得）
  return recur(0, 0, inorder.length - 1);
}

public TreeNode recur(int preRoot, int inLeft, int inRight){
  if(inLeft > inRight) return null;
  //根据根节点索引在前序遍历中查找根节点
  TreeNode root = new TreeNode(preOrder[preRoot]);
  //在哈希表中查询前序遍历中得到的根节点确定根节点在中序遍历中的位置。
  int idx = map.get(preOrder[preRoot]);
  //获取根节点的左子节点。三个参数分别为左子树根节点前序遍历索引，左子树开始位置，左子树的截止位置（idx - 1）中序遍历的根节点索引之前的索引就是左子树节点。
	root.left = recur(preRoot + 1, inLeft, idx - 1);
  //获取根节点右子节点。三个参数分别为右子树根节点前序遍历索引（根节点位置+左子树长度即为右子树的根节点索引），右子树的起始索引，右子树的截止索引。
  root.right = recur(preRoot + (idx - inLeft) + 1, idx + 1, inRight);
  return root;
}

剑指offer26

剑指offer26题主要考察的就是树的深度遍历，当判断一个树是否为另一个树的子结构时需要满足三个条件

  1. 根节点是否相同；

  2. 左子节点是否相同；

  3. 右子节点是否相同；

只要清楚知道上述的三个条件和树的深度遍历思想，这道题就迎刃而解了，先寻找相同的根节点然后再判断左右子节点即可。相同的根节点有可能会在三个位置出现：

  1. A的根节点处；
  2. A的左子树中；
  3. A的右子树中；

下方给出示例代码：

public boolean isSubStructure(TreeNode A, TreeNode B){
  //空值判断
  if(A == null || B == null) return false;
  //判断上述的三个条件，必须同时满足才可以将B称为A的子树
  if(A.val == B.val && recur(A.left, B.left) && recur(A.right, B.right)) return true;
  //寻找相同的根节点，相同的根节点要么在左子树中存在要么在右子树中存在(即使同时存在也不影响），所以此处用||
  return isSubStructure(A.left, B) || isSubStructure(A.right, B);
}

//对所有节点进行逐个判断
public boolean recur(TreeNode A, TreeNode B){
  //B值为null意味着B树中的所有节点都通过了校验
  if(B == null) return true;
  //A值为null意味着A树中所有的节点都不符合要求
  if(A == null) return false;
  //相同时继续分别判断二叉树A，B的左右节点
  if(A.val == B.val) return recur(A.left, B.left) && recur(A.right, B.right);
  return false;
}

剑指offer28

剑指offer28对称二叉树的性质是根节点的左节点等于其右节点，知道这一点后问题就迎刃而解了。从树的根节点开始递归调用，判断其左子节点是否等于右子节点即可。剑指offer27题与剑指offer28题有异曲同工之处，建议一起理解。代码如下：

public boolean isSymmetric(TreeNode root){
  if(root == null) return true;
  return recur(root.left, root.right);
}

public boolean recur(TreeNode left, TreeNode right){
  //当左右子节点都为空时意味着完成了所有节点的校验
  if(left == null && right == null) return true;
  //当有一侧节点先行为空时或左子节点值不等于右子节点值时，即为不对称
  if(left == null || right == null || left != right) return false;
  return recur(left.right, right.left) && recur(left.left, right.right);
}

剑指offer33

剑指offer33中有一个很重要的条件那就是二叉搜索树，所谓二叉搜索树就是根节点以左的节点值都小于根节点，根节点以右的节点值都大于根节点。利用后序遍历左->右->根的顺序即可快速定位到根节点，在利用二叉搜索树的性质进行比较即可。示例代码如下：

public boolean verifyPostorder(int[] postorder){
  return recur(postorder, 0, postorder.length - 1);
}

public boolean recur(int[] postorder, int l, int r){
  //当l大于r的时候，证明校验已经完成
  if(l >= r) return true;
  int p = l;
  //r节点是后序遍历的中的最后一个索引，即为根节点，当前节点大小值小于根节点时移动指针
  while(postorder[p] < postorder[r]) p++;
  //mid值即为第一个大于根节点的节点值，同时意味着从mid节点开始，进入到了根节点的右子树中
  int mid = p;
  while(postorder[p] > postorder[r]) p++;
  //指针移动结束时，索引值等于根节点索引证明所有节点符合二叉搜索树性质，继续判断当前节点的左子树与右子树是否符合
  return p==r && recur(postorder, l, mid - 1) && recur(postorder, mid, r);
}

List<List<Integer>> res = new ArrayList<>();
List<Integer> path = new path<>();
public List<List<Integer>> pathSum(TreeNode root, int target){
  dfs(root, target);
  return res;
}

public void dfs(TreeNode root, int target){
  //递归的终止条件
  if(root == null) return;
  //将当前节点的值放入到path列表中
  path.add(root.val);
  target -= root.val;
  //target等于0时意味着path中所有元素的和等于target值，并且在当前节点的左右子树都为空时（路径已经到末尾，确保最后一个节点是叶子节点），才可以将当前path列表加入到res列表中
  if(target == 0 && root.left == null && root.right == null){
    res.add(new ArrayList(path));
  }
  //后序遍历
  dfs(root.left, target);
  dfs(root.right, target);
 	//回溯
  path.remove(path.size() - 1);
}

剑指offer36

剑指offer-36考察的是将一个二叉搜索树转换成一个排序的循环双向链表，在这里介绍三个概念：

  • 排序链表：节点按小到大的方式排列
  • 循环链表：链表的末尾节点与头节点相连
  • 双向链表：当前节点指向下一个节点的同时，下一个节点也指向当前节点

本题从排序链表和二叉搜索树作为切入点进行解题，如同上一道题介绍的二叉搜索树的性质，我们只需要前序递归二叉树获取最小的左子节点再按照顺序双向连接节点即可，下方给出示例代码：

Node head, prev;
public Node treeToDoublyList(Node root){
  if(root == null) return null;
  recur(root);
  //设置头节点的左指针指向链表的最后一个节点
  head.left = prev;
  //链表的最后一个节点指向头节点
  prev.right = head;
  return head;
}

public void recur(Node curr){
  //递归的终止条件
  if(root == null) return;
  //如果prev是null，证明当前节点是头节点，如果不为空则另上一个节点的右指针指向当前节点
  if(prev != null) prev.right = curr;
  //将头节点的值付给当前节点
  else curr = head;
  //给当前节点赋左指针
  curr.left = prev;
  prev = curr;
  recur(curr.right);
}

  dfs(root.right);
  if(--k == 0) res = root.val;

}

剑指offer55-II

剑指offer55-II主要考察平衡二叉树的问题，关于平衡二叉树的性质，给出定义，如果某二叉树中的任意节点的左右子树的深度相差不超过1，即可将其称之为平衡二叉树。结合剑指offer55-I一起看，这道题无非就是一道求深度的变种题目。代码如下：

public boolean isBalanced(TreeNode root){
  if(root == null) return true;
  //（左子树的深度 - 右子树的深度）的绝对值小于2证明当前树为平衡二叉树，继续判断左子树与右子树
  return Math.abs(depth(root.right) - depth(root.left)) < 2 && isBalanced(root.left) && isBalanced(root.right);
}

public int depth(TreeNode root){
  if(root == null) return 0;
  return Math.max(depth(root.left), depth(root.right)) + 1;

剑指offer32-II与上一道题一样只不过是输出形式不同，这里就不给出代码了。

剑指offer32-III依旧是二叉树的层次遍历问题，要求偶数层从左到右输出，奇数层从右到左输出，下方给出代码。

public List<List<Integer>> levelOrder(TreeNode root){
  Queue<TreeNode> queue = new LinkedList<>();
  List<List<Integer>> res = new ArrayList<>();
  if(root != null) queue.add(root);
  while(!queue.isEmpty()){
    LinkedList<Integer> curr = new LinkedList<>();
    if(int i = queue.size(); i > 0; i--){
      TreeNode node = queue.poll();
      if(res.size() % 2 == 0) curr.addLast(node.val);
      else curr.addFirst(node.val);
      if(node.left != null) queue.add(node.left);
      if(node.right != null) queue.add(node.right);
    }
    res.add(curr);
  }
  return res;
}

2.4 查找公共父节点

剑指offer68-I-二叉搜索树的最近公共祖先

剑指offer68-II-二叉树的最近公共祖先

剑指offer68-I公共父节点可以出现的位置一共有三种情况：

  1. 两个子节点都在左子树中（一个节点是另一个节点的父节点）
  2. 两个子节点都在右子树中（一个节点是另一个节点的父节点）
  3. 两个子节点分散在左右子树中

利用本题中的二叉搜索树条件，可以轻松写出代码：

public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        if(root == null) return null;
        if(p.val > root.val && q.val > root.val) return lowestCommonAncestor(root.right, p, q);
        if(p.val < root.val && q.val < root.val) return lowestCommonAncestor(root.left, p, q);
        return root;
    }

剑指offer68-II题中失去了二叉搜索树这个条件使得判断变得麻烦了起来，但是本质上的思路还是一样的，分别判断三种情况即可，代码如下：

public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q){
  //如果root是p或者q中的任意一个就证明p或者q是共同父节点或分散在两侧；root为null的时候证明p和q不在所查询的范围中
  if(root == null || root == p || root == q) return root;
  //查询p，q是否在左子树中
  TreeNode left = lowestCommonAncestor(root.left, p, q);
  //查询p，q是否在右子树中
  TreeNode right = lowestCommonAncestor(root.right, p, q);
  //left和right都为null的时候证明p和q都不在树中，返回null
  if(right == null && left == null) return null;
 	//left值是在左子树中查询到的p或者q节点，当left为null的时候证明p和q都不在左子树中，直接返回right值即可（此时是情况2）
  if(left == null) return right;
  //right值是右子树中查询到的p或者q节点，当right为null的时候证明q和p都不在右子树中，直接返回left值即可（此时是情况1）
  if(right == null) return left;
  //如果left和right都不为空时，证明在左子树和右子树中分别找到了p和q，此时是情况3，返回当前根节点即可
  return root;
}
